2.  Executive Summary 

Business Purpose 

The SAMMS Global ETL pipeline migrates and modernizes the legacy SAMMS Global ETL process within Microsoft Fabric. It extracts administrative, billing, and clinical reference data from the SAMMS Global SQL Server database and loads it into Fabric using a metadata-driven Medallion architecture (Bronze, Silver, Gold). 

Unlike the legacy C# services, where extraction and transformation logic were tightly coupled, the Fabric implementation separates orchestration, transformation, publishing and auditing into reusable, configuration-driven components. This improves maintainability, scalability, and operational monitoring while preserving the existing business logic. The pipeline supports ten source tables spanning payer, fee schedule, clinical assessment (BAM, COWS), user/site administration, consent, device, and claim status data.

Transformation Logic 

Schema standardization — data type conversion, column ordering, null handling, audit column alignment 

SiteCode resolution — internal location identifiers replaced with business SiteCodes via the Locations reference table 

Duplicate removal — business-key based deduplication using Spark DataFrame operations before Silver write 

Change detection — RowChkSum comparison for PATS Gold tables; configured business-column comparison where no checksum exists 

Upsert processing — insert new records, update changed records, ignore unchanged records 

Active/inactive record handling — records no longer present in source are marked inactive rather than deleted, preserving history
